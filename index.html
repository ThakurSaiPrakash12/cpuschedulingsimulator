CPU Scheduling Algorithms

First Come First Served (FCFS)

How it Works:

  • Processes are executed in the order they arrive in the ready queue
  • Once a process starts executing, it runs to completion
  • No preemption occurs - running process cannot be interrupted

Characteristics:

  • Simple to implement
  • Non-preemptive
  • Fair in terms of waiting time
  • May lead to convoy effect (short process behind long process)

Advantages:

  • Easy to understand and implement
  • Fair for processes with similar burst times
  • No starvation

Disadvantages:

  • Poor performance for processes with varying burst times
  • Average waiting time can be high
  • Not suitable for time-sharing systems

Shortest Job First (SJF)

How it Works:

  • Process with the shortest burst time is selected for execution
  • Non-preemptive version: process runs to completion once started
  • Preemptive version (SRTF): process can be interrupted if shorter job arrives

Characteristics:

  • Can be preemptive or non-preemptive
  • Minimizes average waiting time
  • Requires knowledge of burst times

Advantages:

  • Minimal average waiting time
  • Optimal for minimizing waiting time
  • Good for batch systems

Disadvantages:

  • May cause starvation for longer processes
  • Difficult to predict burst time
  • Overhead of sorting processes

Round Robin (RR)

How it Works:

  • Each process is assigned a fixed time quantum
  • Processes are executed in a circular queue
  • If process doesn't complete within time quantum, it's preempted

Characteristics:

  • Preemptive scheduling
  • Fair allocation of CPU time
  • Time quantum affects performance

Advantages:

  • Fair to all processes
  • No starvation
  • Good for time-sharing systems

Disadvantages:

  • Performance depends on time quantum size
  • Context switching overhead
  • Not optimal for batch systems

Priority Scheduling

How it Works:

  • Each process is assigned a priority
  • Process with highest priority is selected for execution
  • Can be preemptive or non-preemptive

Characteristics:

  • Priorities can be static or dynamic
  • Can be preemptive or non-preemptive
  • Used in real-time systems

Advantages:

  • Flexible priority system
  • Good for real-time systems
  • Can handle varying process importance

Disadvantages:

  • May cause starvation
  • Priority inversion problem
  • Complex to implement

Shortest Remaining Time First (SRTF)

How it Works:

  • Preemptive version of SJF
  • Process with shortest remaining time is selected
  • Running process can be preempted if shorter job arrives

Characteristics:

  • Preemptive scheduling
  • Minimal average waiting time
  • Requires tracking remaining time

Advantages:

  • Minimal average waiting time
  • Good for interactive systems
  • Optimal for minimizing waiting time

Disadvantages:

  • High context switching overhead
  • May cause starvation
  • Complex implementation
